## 1. Product Management — Full Lifecycle

### 1.1 Discovery & Research
Deep user understanding before building anything:

**User Research Synthesis** (thematic analysis methodology):
1. **Familiarization**: Read through ALL data — interviews, surveys, tickets, analytics. Get the landscape.
2. **Initial Coding**: Tag every observation, quote, data point with descriptive codes. Be generous — easier to merge than split.
3. **Theme Development**: Group codes into candidate themes. A theme captures something important about the data relative to the research question.
4. **Theme Review**: Check themes against data. Sufficient evidence? Themes distinct? Coherent story?
5. **Theme Refinement**: Define and name each theme clearly. 1-2 sentence description.
6. **Report**: Write up as findings with supporting evidence.

**Triangulation for Strong Findings**:
- Methodological: Same question, different methods (interviews + survey + analytics)
- Source: Same method, different participant segments
- Temporal: Same observation at different time points
- A finding supported by multiple sources is far stronger. When sources disagree — that's signal, not error.

**Persona Development from Research** (evidence-based, never fictional):
```
[Persona Name] — [One-line description]
Who they are: Role, company type/size, experience level
What they're trying to accomplish: Primary goals, JTBD, success measures
How they use the product: Frequency, key workflows, adjacent tools
Key pain points: Top 3 frustrations, workarounds developed
What they value: What matters most, what would cause churn
Representative quotes: 2-3 verbatim quotes
```

Rules: 3-5 personas max. Behavioral segments > demographics. Update quarterly.

**Jobs-to-be-Done Mapping**:
- Functional jobs (what they're trying to accomplish)
- Emotional jobs (how they want to feel)
- Social jobs (how they want to be perceived)
- Job maturation: Emerging → Competing → Dominant phases

**Opportunity Sizing**:
- Addressable users × Frequency × Severity = Impact score
- Evidence strength: Multiple sources > single source, behavioral > stated
- Use ranges, not false precision: "1,500-2,500 users monthly" not "2,137"

### 1.2 Strategy & Vision
- North star metric selection (the ONE number that best captures value delivery)
- OKR framework: Company → Team → Individual alignment
- 3-horizon roadmapping: H1 (0-3 months, build), H2 (3-12 months, invest), H3 (1-3 years, explore)
- Vision articulation: Where are we going and WHY

### 1.3 Prioritization Frameworks
Apply the right framework to the right decision:

| Framework | Best For | How It Works |
|-----------|----------|-------------|
| **RICE** | Feature prioritization | (Reach × Impact × Confidence) / Effort |
| **MoSCoW** | Release scoping | Must/Should/Could/Won't categorization |
| **ICE** | Quick scoring | Impact × Confidence × Ease |
| **Kano** | Feature classification | Basic/Performance/Excitement categories |
| **Now/Next/Later** | Roadmap communication | Temporal grouping for stakeholders |

### 1.4 Specification (PRD Framework)
Every PRD contains:
```
Problem Statement: What problem, who has it, evidence it exists at scale
User Stories: As [persona], I want [action] so that [outcome]
Requirements: Functional (must-do), Non-functional (performance, security)
Acceptance Criteria: Given/When/Then — unambiguous pass/fail
Success Metrics: Primary metric + guardrail metrics
Out of Scope: What we're explicitly NOT building
Dependencies: What blocks us, what we block
```

### 1.5 Sprint Planning
- Capacity-aware sprint goals (account for ceremonies, oncall, tech debt)
- Story breakdown with dependency mapping
- Velocity tracking with 3-sprint rolling average
- Buffer: 20% capacity reserved for unplanned work

### 1.6 Stakeholder Communication
Tailored by audience:
- **Executives**: Business outcomes, metrics, risks, decisions needed. Weekly/monthly.
- **Engineering**: Technical context, requirements clarity, trade-offs. Per-sprint.
- **Customers**: Value delivered, what's coming, how to use it. Per-release.
- **Cross-functional**: Dependencies, timelines, blockers. Weekly standup format.

## 2. Growth Engineering — AARRR Framework

### 2.1 Acquisition
Multi-channel acquisition with attribution:
- **Organic**: SEO, content marketing, community, word-of-mouth
- **Paid**: Meta/Google/TikTok/LinkedIn ads with ROAS targets
- **Viral**: Referral programs, shareable features, network effects
- **Partnerships**: Co-marketing, integrations, affiliate programs

**Growth Loops** (sustainable, compounding growth):
- Content loop: Create → Rank → Traffic → Users → Create
- Viral loop: User → Invite → New User → Invite
- Product-led: Use → Value → Share → New User → Use
- Community-led: Join → Participate → Create → Attract → Join

### 2.2 Activation
Time-to-value is everything:
- **Aha moment identification**: What action predicts long-term retention?
- **First-run experience (FRX)**: Progressive disclosure, not overwhelming
- **Onboarding optimization**: Measure completion rates per step, identify drop-off
- **Activation rate**: % of signups who reach aha moment within first session/week

### 2.3 Retention Engineering
Habit formation is the moat:
- **Hook Model**: Trigger → Action → Variable Reward → Investment
- **Cadence personalization**: Discover preferred engagement frequency per user
- **Re-engagement**: Behavioral triggers (inactivity), personalized messaging
- **Churn prediction**: Build models on behavioral signals (login frequency, feature usage decline)
- **Cognitive load reduction**: Break 50-item task queue into 1 critical item
- **Micro-sprints**: Time-boxed actions ("5-minute sprint") with momentum building
- **Celebration**: Celebrate micro-wins BEFORE focusing on remaining work

### 2.4 Referral & Virality
- **K-factor optimization**: K = invites_per_user × conversion_rate_per_invite
- **Two-sided incentives**: Both referrer AND referee get value
- **Ambassador programs**: Power users as advocates with exclusive benefits
- **Viral coefficient tracking**: K > 1 = exponential growth

### 2.5 Revenue & Monetization
- **Value metric alignment**: Charge for the unit of value customers perceive
- **Expansion revenue**: Upsells, cross-sells, seat additions
- **Net Revenue Retention (NRR)**: Target >120% for SaaS
- **Rule of 40**: Growth rate (%) + Profit margin (%) ≥ 40

### 2.6 Experimentation
- **A/B test design**: Hypothesis → Variant → Sample size → Duration → Analysis
- **Statistical significance**: Minimum detectable effect, power analysis
- **Guardrail metrics**: What must NOT degrade when testing
- **Experiment velocity**: Ship 2-4 experiments per sprint

## 3. Behavioral Psychology & Nudge Engine

### 3.1 PLFS — Psychological Leverage & Feasibility Score
Proprietary scoring system (1-15 scale) for evaluating which psychological principles to apply:

**Dimensions** (each scored 1-3):
- **Behavioral Leverage**: How strongly does this principle influence the target behavior?
- **Context Fit**: How well does it align with product, audience, and stage?
- **Speed to Signal**: How quickly can we observe impact?
- **Ethical Safety**: Low risk of manipulation or backlash?
- **Implementation Cost**: (subtracted) How hard is it to apply correctly?

**Formula**: `PLFS = (Leverage + Fit + Speed + Ethics) − Implementation Cost`

**Scoring Bands**:
- 12-15: High-confidence lever → Apply immediately
- 8-11: Strong lever → Prioritize in next sprint
- 4-7: Situational → Test carefully with guardrails
- 1-3: Weak → Defer
- ≤0: Risky → Do not recommend

**ALWAYS score before applying any psychological principle.**

### 3.2 Journey-Based Bias Selection
Apply the right psychology at the right stage:

**Awareness Stage**: Mere Exposure, Availability Heuristic, Authority Bias, Social Proof
**Consideration Stage**: Framing Effect, Anchoring, Jobs-to-be-Done, Confirmation Bias
**Decision Stage**: Loss Aversion, Paradox of Choice, Default Effect, Risk Reversal
**Retention Stage**: Endowment Effect, IKEA Effect, Status-Quo Bias, Switching Costs

### 3.3 Cialdini Principles — Applied
- **Social Proof**: "447 teams shipped faster last week" (specific > "thousands")
- **Reciprocity**: Free value before asking (guides, tools, templates)
- **Commitment & Consistency**: Small asks → Medium asks → Large asks
- **Authority**: Expert endorsements, credentials, certifications
- **Scarcity**: Genuine limits on seats, access, or time windows
- **Liking**: Brand personality, community connection, shared values

### 3.4 Nudge Engine Patterns
- **Default bias leverage**: Pre-drafted templates, pre-filled forms, recommended settings
- **Cadence personalization**: Discover channel preference (SMS vs Email vs In-App)
- **Opt-out architecture**: Always provide off-ramps ("Want 5 more minutes or call it?")
- **Variable rewards**: Surprise value keeps engagement unpredictable and exciting
- **Progress visualization**: Progress bars, streaks, completion percentages
- **Gamification**: Points, badges, leaderboards — used sparingly and meaningfully

### 3.5 Pricing Psychology
- **Anchoring**: Show premium tier first, mid-tier feels like a deal
- **Decoy Effect**: Add a third option that makes the target option look better
- **Charm Pricing**: $99 vs $100 (left-digit effect)
- **Risk Reversal**: "30-day money-back guarantee" removes decision anxiety
- **Bundling**: Combined value perception exceeds sum of individual prices
- **Paradox of Choice**: 3 tiers is the sweet spot. More = decision paralysis.

## 4. Pricing Strategy Framework

### 4.1 Value-Based Pricing Hierarchy
```
Customer Perceived Value      ← Understand this first
────────────────────────────
Your Price                    ← Set here
────────────────────────────
Next Best Alternative         ← Know this
────────────────────────────
Your Cost to Serve            ← This is the floor, not the basis
```

### 4.2 Three Core Pricing Decisions
1. **Packaging**: What's included in each tier?
2. **Value Metric**: What customers pay for (users, usage, outcomes)?
3. **Price Level**: How much each tier costs?

### 4.3 Tier Design
- **Good** (entry): Removes friction, limited usage, targets individual users
- **Better** (anchor): Where most customers should land, full core value, best $/unit
- **Best** (premium): Power users/enterprise, advanced controls, premium support

### 4.4 Freemium vs. Free Trial Decision
- **Freemium** works when: Large market, viral/network effects, clear upgrade trigger, low marginal cost
- **Free Trial** works when: Value requires setup, higher price points, B2B evaluation cycles, sticky post-activation

### 4.5 Pricing Research Methods
- **Van Westendorp**: Price Sensitivity Meter (too cheap → too expensive → optimal)
- **Gabor-Granger**: Demand curve at different price points
- **MaxDiff/Conjoint**: Feature packaging decisions
- **Willingness-to-Pay interviews**: Qualitative depth on value perception

### 6.3 Reverse Engineering Viral Content (6-Step Framework)
1. **NICHE ID**: Identify 10-20 top creators in your space
2. **SCRAPE**: Collect 500-1000+ posts for analysis (Apify, Phantom Buster)
3. **ANALYZE**: Rank by engagement rate, extract patterns (hooks, formats, timing)
4. **PLAYBOOK**: Codify repeatable patterns into templates
5. **LAYER VOICE**: Apply "smart friend" principles — specific > vague, short > long, emotion > abstraction
6. **CONVERT**: Bridge from engagement to business results (drive to owned channels)

## 7. Conversion Rate Optimization (CRO)

### 7.1 Page Conversion Readiness Index (0-100)
Score any page before optimizing:

| Category | Weight | What It Measures |
|----------|--------|-----------------|
| Value Proposition Clarity | 25pts | Can visitors understand the offer in <5 seconds? |
| Conversion Goal Focus | 20pts | Is there ONE clear primary CTA? |
| Traffic-Message Match | 15pts | Does the page match what brought them here? |
| Trust & Credibility Signals | 15pts | Social proof, logos, testimonials near CTAs? |
| Friction & UX Barriers | 15pts | Load time, form length, mobile experience? |
| Objection Handling | 10pts | Are common concerns addressed on-page? |

**Readiness Bands**:
- 85-100: High readiness → Run optimization experiments
- 70-84: Moderate → Fix key issues before testing
- 55-69: Low → Foundational problems need solving first
- <55: Not conversion-ready → CRO experiments won't work yet

### 7.2 CRO Diagnostic Order
Always diagnose in this sequence (highest impact first):
1. Value proposition & headline clarity
2. CTA strategy & hierarchy
3. Visual hierarchy & scannability
4. Trust & social proof relevance
5. Objection handling mechanisms
6. Friction & UX barriers

### 7.3 Trust & Social Proof Specifications
- **Specificity beats adjectives**: "447 customers" > "thousands"
- **Relevance to audience**: Enterprise logos for enterprise pages, indie for indie
- **Placement near CTAs**: Social proof within 200px of conversion action
- **Recency**: Fresh testimonials > old ones. Date-stamp reviews.
- **Diversity**: Mix testimonial types — quotes, logos, case study links, ratings

## 8. Go-To-Market & Launch

### 8.1 Five-Phase Launch Architecture
1. **Internal Launch**: Test with team, iron out major issues, build internal champions
2. **Alpha Launch**: External users in controlled way, validate core functionality
3. **Beta Launch**: Scale early access, generate buzz, recruit advocates and testimonials
4. **Early Access**: Leak product details, gather usage data, optional invite batching
5. **Full Launch**: Open self-serve, start charging, announce across all channels

### 8.2 Product Hunt Launch Blueprint
- Build relationships with top hunters 4-6 weeks before
- Optimize listing: compelling tagline, polished visuals, demo video
- Launch day: Respond to EVERY comment (treat as all-day event)
- Direct all traffic back to site for email capture
- Follow up with engaged users within 48 hours

### 8.3 Launch Communication
- **Press release**: Journalistic angle, not feature list
- **Journalist outreach**: Personal, relevant, timely
- **Changelog**: Regular updates signal active development
- **Interactive demo**: Navattic or similar for try-before-you-buy
- **Comparison pages**: Honest differentiation against alternatives

### 8.4 Positioning Strategy
- Category design: Define the category you want to win
- Messaging hierarchy: Primary claim → Supporting points → Proof points
- Competitive differentiation: What you do that NO ONE else does
- Brand voice calibration: Consistent across all touchpoints

## 10. Analytics & Metrics Architecture

### 10.1 Startup Metrics Core
- **MRR/ARR**: Monthly/Annual Recurring Revenue
- **CAC**: Customer Acquisition Cost (blended and by channel)
- **LTV**: Lifetime Value (LTV:CAC target ≥ 3:1)
- **Churn**: Monthly and annual, gross and net
- **NRR**: Net Revenue Retention (target >120% for SaaS)
- **Payback Period**: Months to recover CAC (target <12)
- **Rule of 40**: Growth rate + Profit margin ≥ 40

### 10.2 Funnel Metrics by Stage
- **Awareness**: Impressions, reach, organic traffic, brand search volume
- **Consideration**: CTR, time on page, scroll depth, form starts
- **Conversion**: Signup rate, trial-to-paid, free-to-paid
- **Retention**: DAU/MAU ratio, cohort retention curves, feature adoption
- **Referral**: Referral rate, K-factor, NPS

### 10.3 Analytics Tool Stack
- **Product**: Mixpanel, Amplitude, PostHog — funnels, cohorts, path analysis
- **Web**: GA4, Plausible — acquisition, behavior, conversion
- **Attribution**: Multi-touch models, incrementality testing, media mix modeling
- **Session**: Hotjar, FullStory — heatmaps, recordings, rage clicks
- **Dashboards**: KPI dashboards, executive reporting, weekly metrics reviews

### 10.4 Common Analytics Mistakes
- Reporting averages without distributions (3.5 average could be bimodal)
- Ignoring non-response bias in surveys
- Over-interpreting small differences (0.1 NPS change is noise)
- Treating Likert scales as interval data
- Confusing correlation with causation in cross-tabulations

## Delivery Principles

1. **Problem First**: Validate the problem exists at scale before designing solutions
2. **PLFS Score Everything**: Never apply psychology without scoring for ethical safety
3. **Metric-Led**: Every initiative has a primary metric AND a guardrail metric
4. **Small Bets**: Minimum viable test before full commitment — 2-week experiment sprints
5. **Triangulate**: Qualitative + Quantitative + Business data — never rely on one source
6. **Feedback Loops**: User interviews + analytics + revenue = full picture
7. **Communicate**: Stakeholders informed, aligned, and excited — never surprised
8. **Ship & Learn**: Perfect is the enemy of shipped. Learn from real users faster.
9. **Compound Growth**: Build owned channels. Invest in content loops. Think long-term.
10. **Ethical Growth**: No dark patterns. No manipulative psychology. Users should feel empowered, not tricked.
